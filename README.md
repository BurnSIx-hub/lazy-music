# 🎵 Lazy Music — Foundry VTT

Модуль для Foundry VTT v13/v14, который позволяет GM воспроизводить музыку из **YouTube** прямо в игре, синхронизируя её для всех игроков.

---

## ✨ Возможности

- 🎬 **YouTube** — загрузка плейлистов по URL, поиск треков
- 📡 **Синхронизация** — музыка синхронизируется со всеми игроками через WebSocket
- 🛰️ **Встроенный помощник** — YouTube-треки качаются на компьютер GM и
  раздаются игрокам самим Foundry как обычные аудиофайлы, без YouTube-iframe —
  **полный обход ошибки 150** (встраивание запрещено / VPN). Настроек не требует
- 🎚️ **Контроль воспроизведения** — play/pause/stop/prev/next/shuffle/repeat/volume/seek
- 🔔 **Баннер "Now Playing"** — появляется у всех игроков при смене трека
- 💿 **Мини-плеер** — плавающий диск на экране у всех: клик разворачивает компактную панель.
  У игрока — название трека и личная громкость; у GM — ещё prev/pause/next и мастер-громкость
  «у всех». Перетаскивается за диск, позиция запоминается. Появляется при старте музыки
- 🔍 **Поиск** — поиск треков прямо внутри Foundry
- ⭐ **Свои плейлисты** — собирайте плейлисты из поиска: кнопка «+» на любом YouTube-треке
  добавляет его в свой плейлист (или создаёт новый). Свои плейлисты хранятся в модуле,
  помечены звёздочкой и открываются мгновенно, без обращения к YouTube API. Кнопка «−» убирает трек

## 📦 Установка (для GM)

Требования: Windows, Foundry VTT v13/v14 (десктоп-приложение).
**Игрокам не нужно ничего** — ни модулей, ни настроек: они просто заходят в мир как обычно.

### Способ 1: Через Foundry (рекомендуется)

В Foundry VTT → **Add-on Modules** → **Install Module** → вставьте ссылку на `module.json`
со страницы релизов GitHub (Releases → последний релиз → правый клик по `module.json` → копировать ссылку).

### Способ 2: Вручную

1. Скачайте `module.zip` из последнего релиза на GitHub и распакуйте в:
   ```
   FoundryVTT/Data/modules/lazy-music/
   ```
2. Перезапустите Foundry VTT
3. В настройках мира активируйте модуль **Lazy Music**

### После установки — 3 шага (один раз)

1. **Ярлык.** Дважды кликните `Data/modules/lazy-music/server/create-shortcut.vbs` —
   на рабочем столе появится ярлык **«Foundry VTT (with music)»**. Запускайте Foundry им:
   он стартует Foundry вместе с невидимым музыкальным помощником.
2. **Cookies YouTube.** Поставьте в Chrome расширение «Get cookies.txt LOCALLY»,
   зайдите на youtube.com под своим аккаунтом → иконка расширения → Export.
   Сохраните файл как `%LOCALAPPDATA%\lazy-music\cookies.txt`
   (вставьте этот путь в адресную строку Проводника — папку `lazy-music` создайте сами).
   Без cookies YouTube часто отвечает «подтвердите, что вы не бот» и скачивание не работает.
   Cookies протухают за месяц-два — при ошибках просто переэкспортируйте заново.
3. **Ключ YouTube API** — для загрузки плейлистов и поиска (см. раздел «Настройка» ниже).

## ⚙️ Настройка

### YouTube

1. Перейдите на [Google Cloud Console](https://console.cloud.google.com/)
2. Создайте проект → **APIs & Services** → **Enable APIs**
3. Найдите и включите **YouTube Data API v3**
4. **Credentials** → **Create Credentials** → **API Key**
5. Скопируйте ключ в настройки модуля: **Game Settings → Module Settings → YouTube Data API v3 Key**

> **Примечание:** Бесплатная квота YouTube API — 10,000 запросов/день. Для обычного использования этого достаточно.

### Встроенный помощник (обход ошибки 150)

В папке модуля лежит локальный помощник (`server/`), который скачивает
аудиодорожки YouTube в `cache/` модуля. Раздаёт файлы игрокам **сам Foundry** —
по тому же адресу, по которому игроки уже подключены. Поэтому: никаких настроек,
портов и правил брандмауэра не нужно.

**Как пользоваться:** запускайте Foundry ярлыком **«Foundry VTT (с музыкой)»**
на рабочем столе (он стартует Foundry + невидимого помощника; помощник сам
завершается через ~2 минуты после закрытия Foundry). Всё.

Поведение:
- Первое воспроизведение трека — пауза в несколько секунд (качается), дальше — мгновенно из кэша
- Следующий трек плейлиста подкачивается заранее
- Если помощник не запущен — модуль автоматически играет по-старому через YouTube
  (а если в настройках указан внешний сервер «Моя Музыка» — сначала пробует его)
- Кэш сам ограничивается 1,5 ГБ — старые треки удаляются
- Кнопка 🧹 в шапке окна модуля — ручная очистка кэша (показывает размер, спрашивает подтверждение);
  удалённые треки заново скачаются при следующем воспроизведении
- Кнопка папки в шапке открывает каталог со скачанными треками в Проводнике

**cookies.txt (обязательно один раз):** YouTube без cookies часто требует
«подтвердите, что вы не бот». Экспортируйте cookies расширением Chrome
«Get cookies.txt LOCALLY» (зайти на youtube.com под своим аккаунтом → Export)
и сохраните в `%LOCALAPPDATA%\lazy-music\cookies.txt` (или в `P:\сайт\cookies.txt` —
помощник возьмёт тот файл, что свежее). Протухают за месяц-два — при ошибках
скачивания просто переэкспортируйте.

Технически: помощник — это Deno-скрипт `server/helper.mjs`, слушает только
`127.0.0.1:8766` (снаружи недоступен), качает через автономный `yt-dlp.exe`
из `server/bin/`. Лог — `server/helper.log`. Путь к Foundry прописан в
`server/start-foundry-with-music.vbs` — поправьте, если Foundry установлен в другое место.

## 🎮 Использование

1. Откройте панель Sound Controls (иконка ноты на сцене)
2. Нажмите на иконку **Lazy Music** (♪)
3. Добавьте YouTube-плейлист кнопкой **+** или выберите из уже загруженных
4. Нажмите на трек — он начнёт воспроизводиться
5. Переключатель **Sync** синхронизирует музыку с игроками

## 🔧 Технические детали

| Функция | Реализация |
|---|---|
| YouTube плеер | встроенный помощник (yt-dlp → кэш → раздаёт Foundry); запасной путь — YouTube IFrame API (скрытый iframe) |
| Синхронизация | Foundry WebSocket (`game.socket`) |
| Позднее подключение | Игрок запрашивает у GM текущий трек, позицию, паузу и мастер-громкость |
| Настройки и свои плейлисты | Client settings и localStorage на компьютере GM |

## ❓ Часто задаваемые вопросы

**Q: Слышит ли музыку GM и игроки одновременно?**  
A: Да, при включённом Sync. GM управляет воспроизведением, игроки получают команды через WebSocket и запускают аудио в своём браузере.

**Q: YouTube показывает рекламу?**  
A: YouTube IFrame API воспроизводит видео стандартным способом — реклама возможна на некоторых треках. Для минимизации рекламы используйте треки без монетизации.

**Q: Модуль работает на публичном сервере?**  
A: Да. При использовании встроенного помощника аудиофайлы игрокам раздаёт сам Foundry по обычному адресу мира.
